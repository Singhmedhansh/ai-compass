# v2.1 — Visual Redesign + Motion Design Release

## Anchors

- **Depth tier:** B — visual restyle to editorial aesthetic (homepage look)
- **Motion scope:** medium-heavy (~15-25h)
- **Deadline:** none — quality over speed
- **Total estimate:** 91-145h, ~3-4 weeks of focused sessions (expanded from initial 61-95h after Phase J prework added scope — see Feature additions section)

## Visual aesthetic

Reference: existing homepage (Hero / WizardDemo / CurationDiscipline / SunoStory / FinalCTA), Anthropic.com (editorial pacing), Linear.app (structural clarity), Stripe.com (data + editorial layering).

**Vocabulary to apply across pages:**
- Generous whitespace; sections breathe
- Editorial typography — large display headlines, restrained body
- Gradient surfaces (`bg-elev` panels, `accent-soft` washes, dimensional layering)
- Mockup chrome (the `wizard.ai-compass.in` browser frame is part of the language; can extract as reusable component)
- Calm accent usage — green only for focal positive states (CTAs, success), never decorative
- Editorial pacing — sections build narrative

## Motion design language (medium-heavy)

**In scope:**
- Page-level route transitions (Framer Motion, ~200-300ms)
- Scroll-triggered section reveals (Intersection Observer + Framer variants)
- Hover micro-interactions (card lift, button color transitions, link underlines)
- Stagger animations on lists (tool grids, collection lists, wizard results)
- Loading state choreography (skeleton → content fade)
- Selective parallax / motion-driven hero treatments on tier-1 pages

**Out of scope (defer to v2.2 if ever):**
- Custom cursor effects
- Heavy WebGL / canvas animations
- Auto-playing video backgrounds

**Motion budget:** max 3 motion treatments per page. Prevents over-engineering.

## Feature additions (Phase J expansion)

Scope additions captured during reference-site prework. Each has
rationale + estimated effort.

### Modal login pattern (Lovable-style)
Replace the current bland /login and /register white-page
experience with a modal overlay over the homepage. Page content
behind dims with a frosted backdrop; closing the modal returns
to homepage state with no route change. Routes /login and
/register still exist for SEO/bookmarks; they redirect to
homepage with modal state opened.
- Effort: ~3-5h
- Adds polish; user stays in context
- Replaces tiered Tier-3 LoginPage + RegisterPage redesign
  effort (those pages effectively become the modal; AuthCallback
  stays as a route)

### Number count-up animations on scroll
CurationDiscipline section's 443 / 0 / 38 / 7d figures animate
counting from 0 to target over ~1-1.5s when the section enters
viewport. Apply to any large-figure stats elsewhere if added.
- Effort: ~1-2h
- Library options: react-countup, or hand-rolled with Framer
  Motion's useMotionValue + useTransform

### Interactive wizard preview demo
Replace current static WizardDemo billboard with an interactive
version where users can poke at the wizard inline on the
homepage — fill a question, see preview update — without
navigating away. Acts as a "try before you click" experience
leading naturally into the full /ai-tool-finder.
- Effort: ~6-10h
- Likely shares logic with the real wizard via extracted hooks
  (useFinderRanking, useDebouncedFetch)
- Backend: same /api/v1/finder endpoint, no API changes

### Tool comparison feature
New page /compare?tools=a,b,c (2-4 tools side-by-side).
Row-aligned attributes: name, vendor logo, pricing tier
breakdown, platforms, key features, rating, "best for X"
summary. Entry points: "Compare" button on ToolDetailPage
(adds current tool to compare list), comparison-mode in
DirectoryPage results.
- Effort: ~8-15h
- New route + page component
- Maybe new endpoint /api/v1/tools/compare or compose
  client-side from existing /api/v1/tools/:slug calls

### Tool pricing tiers
Display structured pricing tiers on ToolDetailPage. Each tier
shows: name (Free / Pro / Max etc.), price + billing cycle,
benefits list, limitations list. Comparison view uses the
same data for a pricing row.

Schema extension to tools.json:
  "pricing_tiers": [
    { "name": "Free", "price": 0, "billing": null,
      "benefits": [...], "limitations": [...] },
    { "name": "Pro", "price": 20, "billing": "monthly",
      "benefits": [...], "limitations": [...] }
  ]

Data collection strategy: top-50 most-clicked tools get full
pricing-tier data via AI-assisted population in a SEPARATE dedicated
session (Claude reads tools.json catalog of 443 tools, drafts
pricing for each; Medhansh reviews + commits). Remaining 393 tools
fall back to existing simple "free / freemium / paid" badge. UI
handles both states gracefully.

UI development is INDEPENDENT of data population: schema + components
ship in v2.1 even if only a handful of tools have populated tier
data on day-one. Data collection proceeds in parallel and post-merge
without blocking the v2.1 release.
- Effort: ~7-11h dev (schema + UI + integration)
- Effort: ~5-8h data (separate AI session, can overlap with UI dev
  or run post-merge)
- Hard-rule check: this is TOOL pricing display (product info),
  NOT AI Compass platform monetization. Original "no pricing/
  monetization yet" rule preserved.

## Motion patterns observed (Phase J prework)

Notes captured during reference-site homework. Each pattern has a
mechanics sketch + AI Compass application. These feed directly
into Phase K foundation work (Framer Motion variants library,
scroll hooks).

### anthropic.com

**Pattern 1: Scroll-triggered wordmark → compact mark**
- Top of page: full "ANTHROP\C" wordmark in nav.
- After scroll past hero: collapses to "A\" compact mark.
- Mechanics: scroll position crosses a threshold (~80-120px);
  crossfade between two SVG variants via AnimatePresence;
  ~200-250ms duration; layout-stable (compact mark width matches
  wordmark slot, so no layout shift).
- AI Compass application: "AI Compass" wordmark in Navbar →
  collapses to compass-icon (or "AC" glyph) after scrolling past
  hero on every page. Polish at zero functional cost.

**Pattern 2: Hero card scale-on-reveal**
- Tier-1 hero (Project Glasswing card): image scales up dramatically
  as the section enters viewport.
- Mechanics: pinned-scroll section + `scale()` transform driven by
  scroll progress (Framer Motion's `useScroll` + `useTransform`).
  Image goes from ~0.85x to 1.0x across the pin distance.
- AI Compass application: Tier-1 page heroes — ToolDetailPage's
  hero image, DashboardPage's welcome panel, possibly Best* SEO
  templates. Adds dimensionality without being gimmicky.

### apple.com

**Pattern 3: Mega-menu with frosted backdrop**
- Hover/click on top-level nav category opens multi-column
  dropdown panel
- Page content behind menu dims with a frosted blur
- Three-column structure with section headers + linked items
- ~250-400ms calm easing on expand/collapse
- Mechanics: AnimatePresence on dropdown panel + backdrop with
  backdrop-filter blur + opacity transition
- AI Compass application: probably overkill for current Guides
  dropdown (only 2 entries) but stealable for: theme/avatar
  dropdowns, mobile hamburger menu (frosted backdrop instead of
  solid slide).

**Pattern 4: Restrained nav rest state**
- No icons except brand mark + functional ones (search, bag)
- No underlines on links until hover
- Subtle bg fill behind nav (near-white with thin border)
- Every pixel earns its place
- AI Compass application: audit current Navbar against Apple's
  pixel discipline during Phase L.
